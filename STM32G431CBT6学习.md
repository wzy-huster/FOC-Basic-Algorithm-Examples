# STM32G431CBT6学习

## 模拟外设

### 比较器 COMP

#### ⚙️ 比较器配置项详解 (CubeMX中的配置)

这些配置是在STM32CubeMX中通过勾选和下拉菜单完成的，它定义了比较器工作的所有硬件参数。

| 配置类别         | 具体配置项             | 可选项/说明                                                  |
| :--------------- | :--------------------- | :----------------------------------------------------------- |
| **输入端配置**   | 同相输入               | 可选择**外部GPIO引脚**、**DAC输出通道**或**内部参考电压**的分压（1/4、1/2、3/4）。 |
|                  | 反相输入               | 同样可选择外部GPIO引脚、DAC输出通道或内部参考电压及其分压。  |
| **输出端配置**   | 输出极性               | 选择比较结果是否需要逻辑反转：`Non Inverted`（同相）或 `Inverted`（反相）。 |
|                  | 输出重定向             | 将比较器的数字输出结果连接到内部的不同目的地：<br>- **GPIO**：输出到外部引脚。<br>- **定时器刹车 (Break)**：直接作为高级定时器的刹车输入，实现硬件级快速保护。<br>- **定时器输入捕获**：连接到定时器的输入通道，用于测量脉宽等。 |
| **高级功能配置** | 迟滞 (Hysteresis)      | 用于滤除输入信号噪声，防止输出在阈值点附近频繁抖动。通常有 `No hysteresis`，`Low hysteresis`，`High hysteresis` 等选项。 |
|                  | 消隐 (Blanking)        | 在某些特定时间段内（如PWM输出的“导通”瞬间）屏蔽比较器的输出，防止开关噪声引起的误触发。消隐源通常来自一个定时器的输出比较通道。 |
|                  | 功耗/速度模式          | 在 `High speed` (快速，功耗较高) 和 `Medium speed` (较慢，功耗较低) 之间权衡选择。 |
|                  | 窗口模式 (Window Mode) | 一种特殊模式，需要将两个比较器组合使用，用于检测输入信号是否处于一个预设的电压窗口内。在CubeMX中通常需要成对选择比较器来启用此模式。 |

#### 🛠️ 常用操作函数句柄 (应用代码中使用)

这些是你在应用代码中需要调用的HAL库函数，用于控制比较器和获取结果。

| 功能分类              | 函数原型                                              | 说明                                                         |
| :-------------------- | :---------------------------------------------------- | :----------------------------------------------------------- |
| **启动与停止**        | `HAL_COMP_Start(COMP_HandleTypeDef *hcomp)`           | 启动比较器（非中断模式）。                                   |
|                       | `HAL_COMP_Stop(COMP_HandleTypeDef *hcomp)`            | 停止比较器。                                                 |
|                       | `HAL_COMP_Start_IT(COMP_HandleTypeDef *hcomp)`        | 启动比较器并开启中断。                                       |
|                       | `HAL_COMP_Stop_IT(COMP_HandleTypeDef *hcomp)`         | 停止比较器并关闭中断。                                       |
| **窗口模式启动/停止** | `HAL_COMP_WINDOW_Start(...)`                          | 启动一个窗口比较器组（非中断模式）。                         |
|                       | `HAL_COMP_WINDOW_Stop(...)`                           | 停止一个窗口比较器组。                                       |
|                       | `HAL_COMP_WINDOW_Start_IT(...)`                       | 启动一个窗口比较器组并开启中断。                             |
|                       | `HAL_COMP_WINDOW_Stop_IT(...)`                        | 停止一个窗口比较器组并关闭中断。                             |
| **读取结果**          | `HAL_COMP_GetOutputLevel(COMP_HandleTypeDef *hcomp)`  | **【重点】** 读取比较器当前的输出电平（高/低），是轮询模式下的核心函数。 |
| **中断处理**          | `HAL_COMP_IRQHandler(COMP_HandleTypeDef *hcomp)`      | **【关键】** 比较器中断服务函数。必须在比较器对应的中断服务函数（例如 `COMP1_2_3_IRQHandler`）中调用此函数。 |
|                       | `HAL_COMP_TriggerCallback(COMP_HandleTypeDef *hcomp)` | **【用户实现】** 比较器输出触发中断时的回调函数。用户需要在此函数中编写自己的中断响应逻辑。 |
| **安全锁定**          | `HAL_COMP_Lock(COMP_HandleTypeDef *hcomp)`            | 锁定比较器的配置寄存器，防止其被意外修改，直到下次系统复位才会解锁。适用于安全关键场合。 |
|                       | `HAL_COMP_WINDOW_Lock(...)`                           | 锁定一个窗口比较器组的配置。                                 |

---

#### 💎 总结：一个典型的“中断模式”使用流程

1.  **CubeMX配置**：根据你的需求，配置好比较器的输入、输出、迟滞等所有参数，并勾选 `NVIC Settings` 中的中断使能。
2.  **初始化**：系统初始化代码会自动调用 `HAL_COMP_Init()` 等函数完成硬件配置。
3.  **启动**：在你的应用代码中，调用 `HAL_COMP_Start_IT(&hcomp1);` 来使能比较器和其中断。
4.  **响应**：当比较条件满足时，程序会进入中断，`HAL_COMP_IRQHandler()` 被调用，最终会执行到用户重写的 `HAL_COMP_TriggerCallback()` 函数，你在其中完成业务逻辑。

关于你想了解的 `HAL_COMP_GetOutputLevel()`，它是轮询模式下的核心，可以方便地获取实时比较结果。如果你在做电机控制，`定时器刹车 (Break)` 的输出配置会比输出到 GPIO 更能发挥硬件保护的实时性优势。
